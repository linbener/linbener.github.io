WEB项目开发经验

cons包 公共包

创建的类 成员变量是整个应用需要用到的通用常量

成员变量的类型是 public static final

exception包 异常包

异常类要继承的类有 Exception类，RuntimeException类

Exception类需要提供构造函数，提供错误信息

entry包 实体包

entry包里面的类都需要继承一个基类，而这个基类是实现Serializable接口，并重写了toString方法。

entry类要实现序列化接口，是因为entry也属于持久层，这些类需要在传输层传输，所以要序列化成字符串。

一个基类标准

@transient 只能修饰变量，不能修饰方法和类

java 的transient关键字的作用是需要实现Serializable接口，将不需要序列化的属性前添加关键字transient，序列化对象的时候，这个属性就不会序列化到指定的目的地中。

多对一关系 VS 一对多关系

1 ManyToOne（多对一）

　　单向：不产生中间表，但可以用@Joincolumn（name=” “）来指定生成外键的名字，外键在多的一方表中产生！

2.OneToMany（一对多）

​ 单向：会产生中间表，此时可以用@onetoMany @Joincolumn（name=” “）避免产生中间表，并且指定了外键的名字（别看 @joincolumn在一中写着，但它存在在多的那个表中）

3.OneToMany ,ManyToOne 双向

　　两个注解一起用的）：如果不在 @OneToMany 中加mappedy属性就会产生中间表，此时通常在 @ManyToOne 的注 解下再添上注解 @Joincolumn(name=” “) 来指定外键的名字(说明：多的一方为关系维护端，关系维护端负责外键记录的更新，关系被维护端没有权利更新 外键记录)！（ @OneToMany(mappedBy=”一对多中，多中一的属性”) 出现mapby为被维护端|||默认为延迟加载)

外键在多的一方存在。

dao包 持久层

这个包与数据库交互

关键：创造一个基类，这个基类采用泛型，其他的持久类继承这个类，如此一来，就省去重复写相似代码。
