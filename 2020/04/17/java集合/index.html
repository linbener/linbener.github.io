泛型

泛型需要用 菱形符号表示 <T>

泛型方法既可以存在泛型类当中，也可以存在普通类当中。

对泛型变量进行限定

<T extends Cat> 这个表明泛型T必须是Cat的子类，T的超类可以是Cat，也可以是间接继承Cat的类。

类型擦除

泛型的约束和局限

  1. Java不支持泛型类型的数组
  2. Varags警告 向方法传递一个可变类型的泛型参数就会产生这个警告
  3. 不能实例化泛型对象
  4. ….

通配符类型

<? extends Cat>

通配符的超类型限定

无限定的通配符 ？

在反射中使用的Class类是泛型的

String.class 实际上是一个Class类对象。

泛型类型的继承关系

超类Animal，有一个子类Cat ,而泛型Pets<T>

在Pest<Animal> 之间没有任何的继承关系Pest<Cat>，虽然Cat的超类是Animal。

为了让泛型之间存在继承关系，用上通配符。

Pets<raw> <--- Pest<? extends Animal> <---Pest<Cat> ;

Pest<? extends Animal> <---Pest<Animal>

超类<—子类

Pets <--- Pets<?> <--- Pest<? super Animal> <---Pest<Cat> ;

Pest<? super Animal> <---Pest<Object>

LinkedList

该集合的用到了ListIterator接口，该接口扩展了Iterator接口。

ListIterator接口可以对LinkedList进行顺序和倒叙的遍历，而接口set()更改之前元素的值，它们用在next()之后。 previous()同理。

而ListIterator接口的nextIndex() , previousIndex() 分别返回下一调用next,previous方法的索引，第一次调用的情况下nextIndex返回0，previousIndex返回-1.

LinkedList最好不要使用get()方法遍历，获取元素，因为那样要全部遍历一次集合，效率低下，如果想要遍历某个集合中的元素，应该使用以下的技巧。

接口的add方法能修改集合的结构，可以在集合的任意位置添加元素，比如头部，中间位置，尾部。

数组集合

ArrayList Vector

能用get和set方法随意访问每一个元素，而Vector类的所有方法都是同步的，所以如果不需要同步使用ArrayList。

散列集

对象必须要重写hashCode，equal函数。

填充因子，散列集在填满到一定数量后，自动扩充集合的容量，一般而言填充因子是0.75.

树集

如果填充的对象是自定义的对象，那么需要对象继承Comparable接口。

TreeSet实现了NavigableSet接口，增加反向遍历和定位元素的方法，也可以用NavigableSet接口对TreeSet集合进行新的排序方式。

队列和双端队列

Deque有ArrayDeque 和LinkedList实现，可以根据需要无限扩大容量

优先级队列

优先级有比较产生，有存储的对象实现Comparable接口，或者创建优先级队列时，在构造函数传入一个Comparator对象。

优先级队列 （ priority queue ) 中的元素可以按照任意的顺序插人， 却总是按照排序的顺序进行检索。 也就是说 ， 无论何时调用 remove 方法 ， 总会获得当前优先级队列中最小的元素 。然而， 优先级队列并没有对所有的元素进行排序 。 如果用迭代的方式处理这些元素 ， 并不需要对它们进行排序。

优先级队列使用了一个优雅且高效的数据结构 ， 称为堆 （ heap ) 。 堆是一个可以自我调整的二叉树， 对树执行添加 （ add )和删除 （ remove )操作， 可以让最小的元素移动到根， 而不必花费时间对元素进行排序 。

与 TreeSet—样， 一个优先级队列既可以保存实现了 Comparable 接口的类对象 ， 也可以创建PrrorityQueue构造器中提供的 Comparator 对象。

视图与包装器

轻量级的包装器 Arrays.asList返回的不是集合，而是一个视图对象，只能对视图对象进行get/set操作，任何的改变底层数组的结构的操作，都会抛出UnsupportedOperationException异常。

Collections接口有很多的集合操作方法，不要与Collection接口搞混。

集合 到 子范围视图

集合的 list.subList(start,end) 采用索引确定视图的范围

而对于有序集和映射可以有排序顺序得到子范围。

如有序集合的 subSet headSet tailSet ,Map类似

而Java6引入的NavigableSet接口有更多的确认子范围操作。

不可修改的视图

Collections接口可以得到不可修改值的视图，只能视图进行遍历操作或取值操作，然而产生视图的底层集合依然能够修改。

这些方法很类似 ，如Collections . unmodifiableList
