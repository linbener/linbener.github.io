MySQL是一种数据库管理系统，DBSM。

DBSM分为两种，一是基于共享文件系统的DBSM，一是基于客户机和服务器的DBSM，MySQL是后者。

关系型数据库事务的四大特点：

原子性 一致性 隔离性 持续性

分布式系统的CAP理论 ，对于分布式系统来说，P分区容错性是一定要满足的，所以不同的分布式系统要在A和C之间取舍

C 一致性

A 可用性

P 分区容错性

使用正则表达式进行搜索

REGEXP 用法与LIKE类似

where name REGEXP ‘林..’ ; 匹配name=林XX;

如果匹配的数据是完整的数据时，如 LIKE ‘1000’ ;REGEXP ‘1000’

LIKE 的区别REGEXP，LIKE 匹配的是整个列，如果匹配的文本在列值中出现，LIKE 找不到它，也不会返回；而正则表达式匹配的是列值，所有能返回数据，当然正则表达式也能匹配整个列。

正则表达式一般不区分大小写，如果要区分大小写，要加上关键词BINARY

REGEXP BINARY ‘Jet .00’ ;

事务隔离级别

事务隔离级别 脏读 不可重复读 幻读
读未提交（read-uncommitted） 是 是 是
不可重复读（read-committed） 否 是 是
可重复读（repeatable-read）默认 否 否 是
串行化（serializable） 否 否 否

1、脏读：事务A读取了事务B更新的数据，然后B回滚操作，那么A读取到的数据是脏数据

2、不可重复读：事务 A 多次读取同一数据，事务 B 在事务A多次读取的过程中，对数据作了更新并提交，导致事务A多次读取同一数据时，结果 不一致。

3、幻读：系统管理员A将数据库中所有学生的成绩从具体分数改为ABCDE等级，但是系统管理员B就在这个时候插入了一条具体分数的记录，当系统管理员A改结束后发现还有一条记录没有改过来，就好像发生了幻觉一样，这就叫幻读。

　　小结：不可重复读的和幻读很容易混淆，不可重复读侧重于修改，幻读侧重于新增或删除。解决不可重复读的问题只需锁住满足条件的行，解决幻读需要锁表

设置隔离级别：set session transaction isolation level [serializable…]

查看级别select @@tx_isolation
