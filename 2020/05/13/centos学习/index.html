磁盘管理

类Unix系统的一切文件都是从根”/“ 目录开始，

硬盘设备有很多的扇区组成，其中第一个扇区保存着主引导记录和分区表信息。

单个扇区的容量是512byte组成，主引导记录需要占用446byte，分区表为64byte，而每个分区信息需要用到16byte，所以只能创建主分区不能超过4个

运维人员一般创建3个主分区，外加一个扩展分区。

挂载操作 指的是用户需要使用硬盘设备或分区数据时，需要先将其与一个存在的目录做关联，而这个动作叫“挂载”。

mount 文件系统 挂载目录

逻辑卷管理器：在磁盘分区与文件系统之间添加的一个逻辑层，提供一个抽象的逻辑卷，使得管理者不用担心底层的磁盘布局，对分区的实现更加灵活。

虚拟文件系统VFS：屏蔽真实的文件系统之间的区别

复制文件至其他系统SCP

与远程主机交换文件可是使用FTP，或者搭建WEB服务外，还可以使用SCP, rsync

scp 将本地文件传送至远程主机或从远程主机拉取文件至本地

rsync是Linux系统下常用的数据镜像备份工具，用于不同主机之间同步文件，除了单个文件外，还可以镜像保存整个目录树和文件系统。

Linux防火墙

默认firewalld ；iptables

禁用防火墙

systemctl stop firewalld ; systemctl disable firewalld

systemctl start iptables ;systemctl enable iptables

iptables命令中常见的控制类型有：

ACCEPT LOG记录日志信息，然后传给下一跳匹配规则 REJECT DROP

查看规则：iptables -L

用iptables实现SNAT，即内部地址转换成外部地址，源地址转换技术，让内网用户访问外网

DNAT目的地址转换计划，能让外网IP进入内网

Firewalld

支持图像化工具firewall-config或文本管理工具firewall-cmd

区域概念：

防火墙的网络区域定义了网络连接的可信等级，根据不同的场景调用不同的firewall区域。

区域规则有：

trusted home internal work public external dmz block drop

查看当前的区域：firewall-cmd –get-default-zone

服务的访问控制列表：

允许名单：/etc/hosts.allow

拒绝名单：/etc/hosts.deny

Ngnix服务

Nginx是一款轻量级的WEB服务软件，同时支持负载均衡和反向代理，并发能力强，国内用的居多。

Nginx依赖 gcc ,pcre-devel , zlib-devel ，独立安装，需要安装这些依赖。

因为Ngnix支持https协议，所以还需要安装openssl ,openssl-devel

使用端口80

yum源没有提供ngnix安装包，所以可以换源解决安装问题,Ngnix官网上提供了源，使用这种方式可能会存在一个问题，Failed to start ngnix.service: Unit not found.

解决方案为：添加/etc/init.d/nginx 文件，并赋予权限

service nginx start启动服务

或者用/usr/sbin/nginx 文件的方式启动服务
