创建模式

单例模式

采取一定的方法保证在一个系统中某个类只有一个实例对象。

饿汉式：静态常量，静态代码块，直接用new生成实例对象

懒汉式：线程不安全，同步方法和同步代码块

双重检查法：用双重if判断，内重用同步代码块生成实例对象。

静态内部类：静态内部类某个属性类型是外部类，这个属性用饿汉式的方式生成实例对象。推荐使用

枚举类：构造器私有化，在类的内部实例化一个对象，提供一个方法向外提供实例化对象。推荐使用

工厂模式

简单工厂模式：由一个工厂对象决定创建出的那种类实例对象

工厂方法模式：由定义一个接口，接口内的方法创建实例对象，由实现接口的子类决定创建的类实例对象

抽象工厂模式：简单工厂模式和工厂方法模式的结合

原型模式

用一个类的实例对象创建出另一个新的实例对象。及对象的拷贝。

实现的方式有两个：

  1. 继承cloneable接口，重写clone方法
  2. 用序列化的方式，实现拷贝，及把对象写入输出流后，写入输入流，Java中使用的是对象流，但这种方法可能会很消耗内容。

建造者模式

又叫生成器模式，在工厂模式中有工厂直接生成实例对象，而在建造者模式中它将生成实例对象的过程抽象成为一个抽象类或者抽象接口，建造者拥有这些抽象接口，而建造者调用这些抽象接口的方法，生成具体的实例对象，而建造者不需要知道生成对象的内在逻辑。

四个角色：Builder ConcreteBuilder Product Director

Director拥有Builder接口，而ConcreteBuilder为Builder的具体实现，由它决定生成的Product的种类。

结构型模式

这种模式是通过某些方法将source类实例对象转换成target类实例对象或者在不改变source类的基础上增加target类的功能。

适配器模式

将一个类的接口转换成客户希望的另外一个接口。适配器模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

例如现有两个接口A和接口B互不兼容，如果接口A想拥有接口B的功能，那么必须创建一个中间件，这个中间件必须继承接口A，而且拥有接口B作为成员属性。

所以中间件是一个Adapter类，它继承接口A，拥有某个属性C，这个属性C的类型是接口B，或者接口B的实现类。

中间件类也可以同时继承接口A和继承接口B的实现类。

桥接模式

如果某个对象的创建有诸多的过程，而不同过程之间没有联系，那么就可以把过程抽象出来。

例子：比如画图，具体有两个过程，分别是画一个形状，和给形状上色。它们之间没有只有先后的顺序关系，但没有内在的逻辑关系，比如数据传递。

那么把这两个过程抽象出来，画形状抽象成为一个抽象类，上色抽象成为一个抽象接口，抽象类拥有抽象接口。

给抽象类的某个实现类A赋予某个抽象接口的实现类B，然后实现类A创造出Product。

四个角色：

  • Abstraction：抽象类。
  • RefinedAbstraction：扩充抽象类。
  • Implementor：实现类接口。
  • ConcreteImplementor：具体实现类 。

装饰模式

允许向一个现有的对象添加新的功能，同时又不改变其结构。这种类型的设计模式属于结构型模式，它是作为现有的类的一个包装。

一般有两个主要的角色，装饰者接口和抽象包装者，抽象包装者拥有装饰者接口的成员对象。一般地抽象包装者的构造函数需要传入一个装饰者接口的实现类。

例子：Java中的输入和输出流，各种流的相互包装即为装饰器模式。

组合模式

部分整体模式，是用于把一组相似的对象当作一个单一的对象。组合模式依据树形结构来组合对象，用来表示部分以及整体层次。这种类型的设计模式属于结构型模式，它创建了对象组的树形结构。

组合模式使得用户对单个对象和组合对象的使用具有一致性。

整体是A，那么部分成员是B接口，那么整体A拥有一个集合属性，该属性存放的是B接口的实现类对象。

外观模式

隐藏系统的复杂性，并向客户端提供了一个客户端可以访问系统的接口。这种类型的设计模式属于结构型模式，它向现有的系统添加一个接口，来隐藏系统的复杂性。

关键代码：client拥有抽象接口A，能通过接口A访问不同的子系统，而不同的子系统都继承接口A。

享元模式

主要用于减少创建对象的数量，以减少内存占用和提高性能。这种类型的设计模式属于结构型模式，它提供了减少对象数量从而改善应用所需的对象结构的方式。

享元模式尝试重用现有的同类对象，如果未找到匹配的对象，则创建新对象。

要求：细粒度和共享对象

观察者模式

当对象间存在一对多关系时，则使用观察者模式（Observer Pattern）。比如，当一个对象被修改时，则会自动通知依赖它的对象。

中介模式

是用来降低多个对象和类之间的通信复杂性。这种模式提供了一个中介类，该类通常处理不同类之间的通信，并支持松耦合，使代码易于维护。

备忘录模式

备忘录模式（Memento Pattern）保存一个对象的某个状态，以便在适当的时候恢复对象。

在游戏中非常常见，就是死亡后，通过恢复功能，把主角的数据恢复到某个完整的时刻。

解释器模式

解释器模式（Interpreter Pattern）提供了评估语言的语法或表达式的方式。这种模式实现了一个表达式接口，该接口解释一个特定的上下文。这种模式被用在 SQL 解析、符号处理引擎等。

状态模式

类的行为是基于它的状态改变的。

在状态模式中，我们创建表示各种状态的对象和一个行为随着状态对象改变而改变的 context 对象。

在这种行为模式中，会把一个类的不同状态抽象成为一个类，及把完整的类拆分。

关键代码：一共有两个接口,State接口 和 Context接口，Context接口拥有State接口的作为成员对象，而State接口有一个方法，该方法的接收Context对象，查看State的状态；另一个方法的参数也是Context接口，但这个方法主要是清除Context接口的存储信息。

Context接口的主要功能是 存储 当前的 State接口，并返回此时的 状态。

State接口主要功能是 各种操作，并调用Context接口的方法改变存储的状态。

策略模式

一个类的行为或其算法可以在运行时更改。

在策略模式中，我们创建表示各种策略的对象和一个行为随着策略对象改变而改变的 context 对象。策略对象改变 context 对象的执行算法。

把各种行为抽象成为接口，这些接口要 尽可能相互独立，互不干预。

空对象模式

在空对象模式（Null Object Pattern）中，一个空对象取代 NULL 对象实例的检查。Null 对象不是检查空值，而是反应一个不做任何动作的关系。这样的 Null 对象也可以在数据不可用的时候提供默认的行为。
