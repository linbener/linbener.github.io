JVM学习

自动内存管理机制

Java内存区域和内存溢出异常

堆Head 方法区Method area

虚拟机栈 本地方法栈Stack 程序计数器PC Register

程序计数器，线程当前执行的是一个Java方法，计数器记录的是正在执行虚拟机字节码指令的地址；如果执行的方法是Native方法，则这个计数器为空。

Java虚拟机栈：线程私有，生命周期与线程相同，虚拟机栈描述的是Java方法执行的内存模型，每个方法在执行的同时创建一个栈帧，用于存储局部变量表，操作数栈，动态链接，方法返回信息，附加信息。

虚拟机栈和本地方法栈 都会抛出 StackOverflowError OutOfMemory 异常

Java堆：所以线程共享的内存区域，在虚拟机启动时创建，次内存区域的唯一目的是存放对象实例。

Java堆是垃圾收集器管理的主要区域，因此很多时候被称为GC堆。

由于现在的收集器基本采用 分代收集算法 Java堆还可以细分为 新生代 和 老年代

再细致的有 Eden空间 From Survivor空间 To Survivor空间

方法区：所以线程共享，他用于存储已被虚拟机加载的类信息，常量，静态变量，即时编译器编译后的代码等数据。

运行时常量池：方法区的一部分。Class文件，存放编译期生成的各种字面量和符号引用。

直接内存并不是虚拟机运行时数据区的一部分，也不是Java虚拟机规范中定义的内存区域，这部分被频繁地使用也能导致OutOfMemory 异常。

造成 OutOfMemory 异常：

堆溢出，-Xms -Xmx

栈溢出，-Xss

运行时常量池溢出，-XX: PermSize -XX:MaxPermSize

直接内存溢出 -XX: MaxDirectMemorySize

对象的创建

对象的内存布局，在HotSpot虚拟机中对象在内存中的存储布局分为3块区域，分别是对象头，实例数据和对齐填充。

对象的访问：Java程序需要通过栈上的reference数据来操作栈上的具体对象。reference类型在Java虚拟机规范中是一个指向对象的引用。

Reference类型 具体实现有两种方式：一种是 使用句柄 另一种是 直接指针。

使用句柄：Java堆中划分出一块内存作为句柄池，reference中存储的就是对象的句柄地址，而句柄中包含了对象实例数据和类型数据的地址信息。

直接指针：Java堆需要考虑如何放置访问类型数据的相关信息，而reference存储的是对象地址。

垃圾收集器与内存分配策略

对象已死吗？

引用计数算法：每个对象都有一个引用计数器，每当一个地方引用它，计数器加1，引用失效，计数器减去1. 但引用计数算法有缺陷，

可达性分析算法：通过一系列的称为GC Root的对象为起始点，从这些节点开始向下搜索，搜索所走过的路径称为 “引用链” ，当一个对象到GC Root没有任何的链相邻，从图来说就是从GC Root到这个对象不可达。

在可达性分析中对象要经过两次的标记过程后，才会被垃圾回收器回收，第一次标记之后对象还是有机会拯救自己的。

可以作为为GC Root对象的有：

  1. 虚拟机栈中引用的对象
  2. 方法区中类静态属性引用的对象
  3. 方法区中常量引用的对象
  4. 本地方法栈中JNI引用的对象

引用的概念：

引用不再是reference这个侠义的概念，Java扩展了引用，分为强引用，软引用，弱引用，虚引用。强引用是new 生成的对象，不会被垃圾收集器回收，软引用在第二次回收中回收，弱引用在第一次回收中回收。

垃圾收集算法

标记-清除算法：最基础的算法，其他的算法都是基于改进这个算法延申的，这个算法分为两个阶段，标记阶段标记所以需要回收的对象，清除阶段回收标记对象的内存空间。

标记-清除算法会造成很多不连续的空间碎片。

复制算法：将可用的内存容量划分为大小相等的两块，每次只使用其中的一块。当这一块的内存使用完了，就将其他存活的对象复制到另一块上面，然后清除已使用过内存空间的这一块。

但由于大多数新生代的对象都是朝生夕死，所以一般是将内存空间分为3块，1块是大区域的Eden空间，和2块较小的Survivor空间，每次只使用Eden空间和1块Survivor空间。Eden:Survivor=8:1

标记-整理算法: 根据老年代对象提出的算法，标记阶段和标记-清除算法的第一步相同，整理阶段是所有存活的对象移动到内存空间的一端，然后清除掉另一端以外的内存。

分代收集算法：根据对象的存活周期的不同将内存划分为几块，Java堆一般分为新生代和老年代，根据不同对象使用以上的三种算法。新生代使用复制算法，老年代使用标记-清除或标记-整理算法。

HotSpot算法实现

枚举根节点：根节点及GC Root，在枚举根节点这个阶段所有的线程都要暂停，这个现象被称为 the world stop。Java虚拟机中OopMap这个数据结构存储着所有的对象引用。

安全点：程序执行时并非在所有地方都能停顿下来开始GC,只有到达安全点时才能暂停。安全点选定标准，是否具有让程序长时间执行的特征。

考虑的问题：怎么让所有线程跑到最近的安全点再停顿下来，方案：抢占式中断，主动式中断。

安全区域：解决程序不执行的时候，及某些线程处于sleep状态，或者阻塞状态，不能响应中断信号。安全区域是指一段代码片段中，引用关系不会发生变化。在这个区域中的任意地方开始GC都是安全的。

垃圾收集器：

新生代：Serial ,ParNew ,Parallel Scavenge

老年代：CMS,Serial Old ,Parallel Old

两者都可以的：G1

GC日志：GC日志的开头“[GC 和 “[Full GC，说明了这次垃圾收集的停顿类型，后者说明了GC发生了Stop-The-World.

内存分配与回收策略

  1. 对象优先进入Eden分配内存
  2. 大对象直接进入老年代
  3. 长期存活的对象进入老年代

类文件结构

Class文件是一组以8位字节为基础的单位二进制流

u1,u2,u4,u8 无符号数是基本的数据类型，其中u1代表一个字节的无符号数

表是以_info为结尾的复合数据类型

1魔数：class文件的头4个字节，用来识别class文件。值为 0xCAFEBABE

第5，6字节是次版本号，第7，8字节是主版本号，版本号是从45开始的。

2常量池

魔数之后就是常量池

常量池：字面量和符号引用(类和接口的全限定名，字段的名称和描述符，方法的名称和描述符)

3访问标记

两个字节的访问标记access_flags,用来识别一些类或者接口层次的访问信息

4类索引，父类索引，接口索引集

5字段表集合

6方法表集合 类构造器<clinit> 实例构造器<init>

7属性表集合
